# MDM → ML-HUGS Pipeline: Complete Step-by-Step Guide

## The Problem You Mentioned (Upside-Down Avatar)

This occurs due to **coordinate system mismatch**:
- **MDM**: Uses Y-up convention (standard for motion capture)
- **HUGS**: Uses a different convention (likely Z-up)

## Solution Overview

We convert MDM output with coordinate transformation to fix the orientation.

## Understanding the Coordinate System Options

| Transformation | Effect | When to Use |
|---|---|---|
| `--rotate_x 90` | Swaps Y↔Z, keeps X | Y-up (MDM) → Z-up (common) |
| `--rotate_x 180` | Flips Y and Z | Y-up (MDM) → Inverted Z-up |
| `--rotate_x -90` | Opposite of 90° | Alternative Z-up convention |
| `--rotate_y 90` | Swaps X↔Z, keeps Y | Less common, use if above don't work |
| `--rotate_z 90` | Swaps X↔Y, keeps Z | Rotation in horizontal plane |

---

## Common Issues & Solutions

### Issue: Avatar Still Upside-Down
**Solution**: Try a different rotation parameter
- If `--rotate_x 90` doesn't work, try `--rotate_x 180`
- Or try `--rotate_x -90`

### Issue: Avatar Sideways or Rotated
**Solution**: Try Y or Z rotation combinations

### Issue: Motion Looks Correct But Scaled Wrong
**Solution**: Currently not handled - would need to scale translation vector. Contact if needed.

## Complete Example Walkthrough

```bash
# Set variables
MDM_DIR="/home/sigma/skibidi/motion-diffusion-model"
HUGS_DIR="/home/sigma/project_avatar2_hugs/ml-hugs-NTHUavatar"
SAMPLE_NAME="samples_humanml_enc_512_50steps_000750000_seed10_a_person_jumps"
SAMPLE_DIR="${MDM_DIR}/save/humanml_enc_512_50steps/${SAMPLE_NAME}"

# Step 1: Check if results.npy exists
ls "${SAMPLE_DIR}/results.npy"

# Step 2: Test all variants
cd "$MDM_DIR"
python test_coordinate_fixes.py \
    --mdm_results "${SAMPLE_DIR}/results.npy" \
    --output_dir "./test_conversions_jump"

# Step 3: Try each in HUGS (run individually, check output video each time)
cd "$HUGS_DIR"
for variant in original 90deg_x 180deg_x neg90deg_x 90deg_y 90deg_z; do
    echo "Testing: $variant"
    python main.py --motion_file "${MDM_DIR}/test_conversions_jump/${variant}.npz"
    echo "Check output/human_scene/neuman/... for ${variant}.npz"
    sleep 2
done

# Step 4: Once you know correct variant (e.g., 90deg_x), use it
cd "$MDM_DIR"
python mdm_to_hugs_pipeline.py \
    --mdm_results "${SAMPLE_DIR}/results.npy" \
    --output "${SAMPLE_DIR}/hugs_motion.npz" \
    --rotate_x 90.0

# Step 5: Final render in HUGS
cd "$HUGS_DIR"
python main.py --motion_file "${SAMPLE_DIR}/hugs_motion.npz"
# Output: output/human_scene/neuman/[model]/[config]/[timestamp]/anim_neuman_....mp4
```

## Key Files Created

1. **mdm_to_hugs_pipeline.py** - Main conversion script with coordinate transformation
   - Use this for production once you know the correct transformation

2. **test_coordinate_fixes.py** - Tests all common coordinate transformations
   - Use this first to find the right parameters

3. **sample/extract_smpl_params.py** - Extracts SMPL parameters from MDM output
   - Low-level tool, usually not needed directly

## Notes

- The rotation parameters are applied to the entire coordinate frame
- All joint rotations and translation vectors are transformed together
- Make sure to use consistent parameters across all your conversions
- The most likely fix is `--rotate_x 90.0` for Y-up to Z-up conversion
