# Quick Start: Convert MDM Output to HUGS

## TL;DR - Three Command Walkthrough

Assuming you have a MDM-generated `results.npy` at this path:
```
/home/sigma/skibidi/motion-diffusion-model/save/humanml_enc_512_50steps/samples_humanml_enc_512_50steps_000750000_seed10_a_person_jumps/results.npy
```

### Step 1: Test All Coordinate Transformations (2-3 minutes)
```bash
cd /home/sigma/skibidi/motion-diffusion-model

python test_coordinate_fixes.py \
    --mdm_results save/humanml_enc_512_50steps/samples_humanml_enc_512_50steps_000750000_seed10_a_person_jumps/results.npy \
    --output_dir ./test_conversions
```

This generates multiple `.npz` files in `./test_conversions/`:
- `original.npz` - No rotation
- `90deg_x.npz` - 90° around X (most common fix!)
- `180deg_x.npz` - 180° around X
- `neg90deg_x.npz` - -90° around X
- `90deg_y.npz`, `90deg_z.npz`, etc.

### Step 2: Test Each in HUGS (30 seconds per variant)
```bash
cd /home/sigma/project_avatar2_hugs/ml-hugs-NTHUavatar

# Try this first (most common fix for upside-down)
python main.py --motion_file /home/sigma/skibidi/motion-diffusion-model/test_conversions/90deg_x.npz

# Check output/human_scene/neuman/.../anim_neuman_*.mp4
# If avatar is correct orientation, stop here! Otherwise try next:

python main.py --motion_file /home/sigma/skibidi/motion-diffusion-model/test_conversions/180deg_x.npz

# Try others if needed...
```

### Step 3: Use the Correct One for Production
Once you find the correct orientation (e.g., `90deg_x`):

```bash
cd /home/sigma/skibidi/motion-diffusion-model

# Convert with the correct rotation
python mdm_to_hugs_pipeline.py \
    --mdm_results save/humanml_enc_512_50steps/samples_humanml_enc_512_50steps_000750000_seed10_a_person_jumps/results.npy \
    --output save/humanml_enc_512_50steps/samples_humanml_enc_512_50steps_000750000_seed10_a_person_jumps/hugs_motion.npz \
    --rotate_x 90.0
```

Then in HUGS:
```bash
cd /home/sigma/project_avatar2_hugs/ml-hugs-NTHUavatar

python main.py --motion_file /home/sigma/skibidi/motion-diffusion-model/save/humanml_enc_512_50steps/samples_humanml_enc_512_50steps_000750000_seed10_a_person_jumps/hugs_motion.npz
```

## What's the Problem?

Your upside-down avatar happened because:
- **MDM** uses **Y-up** coordinate system (standard in motion capture)
- **HUGS** expects **Z-up** or different orientation
- Simply passing raw parameters doesn't transform coordinates

## What These Scripts Do

1. **`test_coordinate_fixes.py`** - Tests common rotations automatically
   - Creates multiple versions with different coordinate transformations
   - Helps you find the right one without guessing

2. **`mdm_to_hugs_pipeline.py`** - Converts MDM → HUGS with transformation
   - Handles 6D rotation format conversion
   - Applies coordinate frame rotation
   - Outputs in HUGS-compatible format

3. **`mdm_to_hugs.sh`** - Bash wrapper for quick commands
   - Optional convenience tool

---

## Most Likely Answer

For typical Y-up (MDM) → Z-up (HUGS) conversion, use:
```bash
--rotate_x 90.0
```

This is what you should try first if your avatar is upside-down!

---

## If That Doesn't Work

Try these in order:
1. `--rotate_x 180.0` (flips Y and Z)
2. `--rotate_x -90.0` (opposite direction)
3. `--rotate_y 90.0` (less common)
4. `--rotate_z 90.0` (rotation in horizontal plane)

---

## Verify Success

After running in HUGS, check:
- Avatar stands upright (not upside-down) ✓
- Head points up ✓
- Motion follows gravity (jumps go up then down) ✓
